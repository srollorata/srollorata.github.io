## What I do

Track and manage various project state elements:

### TODO Items
- Identify incomplete tasks and action items
- Track implementation todos in code comments
- Prioritize based on dependencies and urgency
- Update status as work progresses

### Issues
- Catalog bugs, feature requests, and enhancements
- Track issue status (open, in-progress, resolved)
- Link issues to relevant code locations
- Assess severity and impact

### Project Milestones
- Track feature releases and version milestones
- Monitor progress toward goals
- Identify blockers and risks
- Generate milestone status reports

### Technical Debt
- Identify areas of suboptimal implementation
- Track refactoring needs and priorities
- Estimate impact on future development
- Document remediation suggestions

## When to use me

Use this when:
- Starting a new feature or task
- Tracking progress on ongoing work
- Reviewing project health and status
- Planning sprints or releases
- Conducting retrospectives
- Managing technical debt

Provide context about what aspect of project state needs attention, and ask clarifying questions if tracking systems or conventions need to be established.
